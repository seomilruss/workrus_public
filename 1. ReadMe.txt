내 코드의 방향성
내코드 장단점
내 코드의 장점은 개발규모가 작든 크든 정교하게 만들려고 이것 저것 시도해본다. 그래서 코드가 1번 완성이 되면 이전에 개발한 것과 유사한 개발을 할때
몇줄의 코드를 수정만 하면 재사용이 가능하다. 단점은 정교하게 만드는 것을 추구해서 코드의 구조가 복잡해지는 경우가 생겨서
유지보수성이 줄어들 수는 있다.

자가진단
개발규모에 따라 코드의 정교함을 줄이고 단순하게 만들어서 유지보수성을 늘리는 것도 필요하다.

Project VendingMachine(Java Programming_Level1) // 자판기 프로젝트(자바 프로그래밍 등급1)
---------------------------------------------------------------------------------------------------------------------------------------------
▼ account for project :
현실의 자판기 객체를 모델링해서 자바코드로 변환.
#01. 메소드의 활용 : (
	메소드를 활용해서 값과 출력UI와 로직을 구획화했다.  -> 코드 정리 & 유지보수성 향상
)
#02. 내부객체의 활용 : (
	구획화된 UI메소드와 로직 메소드를 Inner객체(DisplayUserInterface 와 MainControler)로 가볍게 감싸주고
	객체타입 변수를 작성해서 has-a 관계로 만들었다. -> 자판기 객체 내부에 부품객체를 가지는 것을 표현.
)
#03. 객체의 정교함을 증가 : (
	VendingMachine 객체 1개만 만들어도 되지만 부가적으로 음료수와 UI와 main제어장치를 만들었다.
	시간낭비일 수도 있지만 1번 만들어두면 나중에 코드가 필요하면 몇줄의 코드만 수정해서
	정교하게 만들어진 객체를 재황용이 가능하고 코드의 신뢰성을 향상가능하다.
	그것을 위한 시간투자다.
	언젠가 음료수의 객체가 필요하면 음료수를 마시는 메소드를 활용해서
	음료수를 마시는 것 까지 이미 구현되어 있다.
	몇 줄의 코드를 수정해서 어떻게 음료수를 마시는지 코드를 추가해주면 동작이 된다.
)
